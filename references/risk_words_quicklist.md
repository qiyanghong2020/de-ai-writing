# 高风险词速查表

这份表给的是改稿时的快速扫词清单。  
不是禁词表，也不是一见就删。用法只有一个：

**如果这个词在句子里只是撑抽象感、撑结构、撑“像在分析”，就优先换成更具体的中文。**

## 1. 滑词

- `落到` -> `变成` / `体现在`
- `大面上` -> `整体看`
- `这一层` -> 直接写具体层面，或删掉
- `底色` -> 直接写起点 / 风险 / 基调
- `摆了出来` -> `写了出来` / `列了出来`
- `接住了` -> `回应到了` / `处理到了`
- `往前看` -> `后面看` / `接下来`
- `更吃力的部分` -> `更难的部分` / `更费工夫的部分`

## 2. 翻译腔抽象词

- `线索` -> `事情` / `方向` / `证据`
- `视角` -> `角度`
- `路径` -> `做法` / `过程`
- `框架` -> `结构` / `规则`
- `图景` -> `情况` / `局面`
- `层面` -> `上`
- `维度` -> `方面`
- `机制` -> `原因` / `过程`
- `叙事` -> `讲法` / `说法`
- `抓手` -> `办法` / `着力点`

## 3. 模板点评词

- `这一点很关键` -> 删掉，直接说关键事实
- `最值得记住的是` -> 直接写结论
- `信息很清楚` -> 直接写清楚的那件事
- `边界也很明确` -> 直接写限制
- `这更像是在提醒我们` -> 直接写这项研究没看到什么 / 说明什么

## 3.5 对比句模板

- `不是……而是……` -> 能直接说就直接说
- `并非……而是……` -> 去掉反驳架子，保留事实
- `不只是……而是……` -> 看情况改成 `同时`
- `不靠……而是……` -> 直接写真正使用的依据或方法

## 3.6 语气撑场词

- `真正` -> 多数时候直接删
- `真正的` -> 多数时候直接删
- `实际上` -> 多数时候直接删
- `最需要` -> 看情况删掉前半句，直接落事实
- `最该` -> 看情况改成平视表达

## 4. 模型默认客观起手

- `研究表明` -> `这项试验里`
- `数据显示` -> `这组数据里`
- `结果提示` -> `图里能看到`
- `研究结果显示` -> `随访结束时`
- `相关分析发现` -> `分析后看到`

## 5. 绕行动词

- `进行了讨论` -> `讨论了`
- `实现了优化` -> `优化了`
- `完成了分析` -> `分析了`
- `做出了判断` -> `判断了`
- `给出了提示` -> `提示`
- `提供了一种解释` -> `解释了`
- `形成了支持` -> `支持了`

## 6. 抽象起手式

- `第一条线索是` -> `先看第一件事`
- `其中一个重要方面是` -> `这里还有一件事`
- `在这个意义上` -> 删掉或直接说结论
- `在这一点上` -> `这里`
- `在这个框架下` -> `按这套规则看`
- `从这个角度出发` -> `从这里看`
- `沿着这个思路` -> `顺着这个问题往下看`

## 6.5 读者带读和身份预设

- `读这篇文章时` -> 直接写内容
- `放在一起看` -> 直接写并列后的结果
- `再往下看` -> 直接写下一层结果
- `最需要盯住的是` -> `更该注意的是`，或直接删掉前半句
- `普通人最该记住的是` -> 直接写结论或行动
- `普通读者` -> 多数时候删掉或改成更中性的对象表达
- `别往后拖` -> `需要尽快处理`
- `不能轻轻放过` -> `需要尽快处理`

## 7. 连接词用太多

- `此外` -> 很多时候直接删
- `然而` -> 如果只是形式转折，很多时候直接删或改成更具体的转折内容
- `总之` -> 多数时候直接落结论，不必先搭总结牌子
- `同时` -> 看情况删掉或改成具体动作
- `另一方面` -> `还有`
- `总的来看` -> `整体看` 或直接说结论
- `值得注意的是` -> 直接写那件事
- `需要指出的是` -> 直接写限制
- `进一步来看` -> 直接写下一层结果
- `从某种程度上` -> 删掉
- `在这一层面` -> 删掉或改具体对象

## 8. 模板化概念块

- `同一套数据治理和评价规则` -> 看情况拆成更具体的 `数据治理办法` / `评价标准`
- `统一表示方法` -> `能把这类数据放进同一套表示里处理的方法`
- `可复用、可核对的证据` -> `后面还能复用、也能对照核查的证据`
- `缺失语义和无序输入一起处理好` -> `既能看懂缺失本身的含义，也能处理没有固定顺序的输入`
- `系统分析` -> 看情况改成 `系统研究` / `放在一起分析`

## 9. 结构性风险

- 连着两段都按同样的综述节奏推进 -> 改变其中一段的任务
- 每段都“铺垫 -> 解释 -> 收尾” -> 让一段直接落问题或结论
- 句子总是把逻辑收满 -> 删掉最后那层重复解释
- 每句都同样完整、同样正式、同样平衡 -> 保持正确，但把句长、句法重量和落点拉开
- 过渡词一多就像讲义 -> 能删就删，不能删就把逻辑关系直接写出来

## 10. 轻度人味，可用但别刷屏

适合少量加：

- `我更在意的是`
- `我会这样理解`
- `其实`
- `说实话`
- `有点可惜`

不适合频繁加：

- `我觉得`
- `说白了`
- `这事儿`
- `你会发现`

## 使用顺序

改稿时建议按这个顺序扫：

1. 先扫模板点评词
2. 再扫滑词和翻译腔抽象词
3. 再扫绕行动词
4. 再扫重复的客观起手和连接词
5. 最后判断要不要补一点轻度人味
