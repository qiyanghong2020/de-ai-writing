# 常见改写对照

这份对照表给的是“常见坏味道 -> 更自然写法”的快捷替换思路。  
不要机械替换，先看上下文是不是同一个意思。

## 1. 点评腔 -> 直接说内容

- `这一点很关键。` -> 直接删掉，下一句直接写关键事实
- `这更像是在提醒我们……` -> 直接写“这项研究没有看到…… / 这一步只说明……”
- `最值得记住的是……` -> 直接写结论
- `信息很清楚。` -> 直接写清楚的那件事
- `边界也很明确。` -> 直接写这项研究没回答什么

## 1.5 对比句先上架子 -> 直接陈述

- `这篇研究不是在回答疗效，而是在回答流程速度。`
  -> `这篇研究回答的是流程速度。`
  -> 如果上下文需要保留对比，也可以写成 `这篇研究看的是流程速度，不是疗效。`

- `真正重要的不是报告更快，而是后面有没有更快确诊。`
  -> `更重要的是，后面有没有更快确诊。`
  -> 如果结果已经明确，也可以直接写 `报告更快了，但后面的确诊没有明显提速。`

- `这项研究不只是看了准确率，而是还看了安全性。`
  -> `这项研究同时看了准确率和安全性。`

- `它不靠主观评价，而是用真实世界结局来判断。`
  -> `它用真实世界结局来判断。`

## 1.6 破折号 `——` 堆叠 -> 用句号和平实标点

AI 味中文很容易把 `——` 当万能连接符：用来定义、补充、举例、总结、过渡，都用这一个符号。密集堆叠后节奏会变得很一致、很"被管理"。规则是：保留真正的转折 / 插入 / 停顿，其它改掉。

- `A——B` 用来重命名或下定义
  -> `A，即 B` / `A：B` / 直接拆成两句 `A。B 是……`
  -> 如果 B 只是一个很短的同位成分，逗号就够了：`A，一个 B`

- `A——因为 B` / `A——它 B` 用来拉长解释
  -> 直接句号拆：`A。因为 B……` / `A。它 B。`
  -> 让上一句先落地，再另起一句把原因写清楚

- `A——B——C` 在一句话里堆三个破折号
  -> 拆成两到三句短句，或者把中间段落改成短定语 / 状语
  -> 保留的那处 `——` 承担一个真正的插入或转折

- `这项研究的意义是……——它做了 X、Y、Z`
  -> `这项研究的意义是……。它做了 X、Y、Z。`
  -> 列表不需要通过 `——` 接出来

- `句尾 ——最后 / 结果 / 也就是……`
  -> 删掉破折号，让上一句自己收口
  -> 如果后半段分量够，另起一句独立表达；不要用破折号当点睛小尾巴

- 相邻段落反复 `——`，形成段落节奏标记
  -> 改掉至少一段，让段落开头、结尾的节奏彼此不同
  -> 能用句号 + 独立短句解决的，就不要让 `——` 来背

诊断线：

- 同一段里超过两处 `——` 基本一定要改一处
- 连续两句都用 `——` 的，至少改一句
- 300 字左右的段落里 `——` 超过 4 处，就全段扫一遍

保留 `——` 的合法场景：

- 真正的插入语或旁注（两侧内容在主干之外）
- 明确的停顿和强调（不是补说明，是刻意断开）
- 原文本身就用 `——`，而且承担的是上述两类作用

## 2. 英文抽象骨架 -> 具体中文

论文 / 医学 / 学术正文先不要机械套这里的替换。  
这类文本里，`框架`、`机制`、`层面`、`维度` 有时是术语或方法概念，应先判断它是不是在承担精确含义。

- `三条线索` -> `三件事` / `三个方向` / `三部分证据`
- `从这个视角出发` -> `从这里看` / `换个角度看`
- `这条路径` -> `这种做法` / `这个过程`
- `在临床层面` -> 如果只是空泛架子，改成更具体的对象，如 `在临床应用中` / `在临床结局上`；如果本来就是精确表述，可保留
- `从多个维度来看` -> 如果维度是实打实的分析维度，直接写出这些维度；如果只是空泛起手，再改成 `从几个方面看`
- `这一图景` -> `这种情况` / `这个局面`
- `这个框架` -> 如果是实际的方法或分析框架，直接点名保留；如果只是抽象支架，改写整句，把设计、步骤或标准写出来

## 3. 绕行动词 -> 直接动词

- `进行了分析` -> `分析了`
- `开展了研究` -> `研究了`
- `进行了讨论` -> `讨论了`
- `进行了说明` -> `说明了`
- `进行了探讨` -> `探讨了`
- `实现了提升` -> `提高了`
- `实现了优化` -> `优化了`
- `完成了分析` -> `分析了`
- `做出了判断` -> `判断了`
- `作出了回应` -> `回应了`
- `形成了支持` -> `支持了`
- `提供了一种解释` -> `解释了`
- `给出了提示` -> `提示`

## 4. 模型默认“客观起手” -> 去重，但别改坏证据类型

论文 / 医学 / 学术正文里，这一节的目标是减少重复起手，不是把证据报告词改成口头观察句。

- `研究表明……` -> 如果对象已明确，可写 `该研究显示……` 或直接写结果；不要把 `研究` 一律改成 `试验`
- `数据显示……` -> 如果数据来源已明确，可直接写结果，或写 `该数据表明……`
- `结果提示……` -> 只有在确实要保留推断层级时再用 `提示`；不要降成 `图里能看到……`
- `研究结果显示……` -> 可以把时间点、组别或终点提前，但保留原有证据性质
- `相关分析发现……` -> 可写 `相关分析显示……` 或直接写分析结果；不要改成 `分析后看到……`

如果上下文已经交代过是谁做的、看了什么，就别每句都再起手一次。

## 5. 滑词 -> 明确动作或结果

- `结果落到……` -> `结果变成了……` / `结果主要体现在……`
- `大面上……` -> `整体看……`
- `接住了这个问题` -> `回应到了这个问题`
- `摆了出来` -> `写了出来` / `列了出来`
- `往两边分开` -> `分成两组`
- `更吃力的部分` -> `更难的部分` / `更费工夫的部分`

## 6. 抽象起手式 -> 更像人说话

- `第一条线索是……` -> `先看第一件事`
- `其中一个重要方面是……` -> `这里还有一件事`
- `在这个意义上……` -> 删掉，直接说结果
- `在这一点上……` -> `这里……`
- `沿着这个思路……` -> `顺着这个问题往下看……`

## 7. 讲义连接词 -> 自然推进

- `此外，……` -> 很多时候直接删掉
- `另一方面，……` -> 看情况改成 `还有……`
- `总的来看，……` -> `整体看……` 或直接说结论
- `值得注意的是，……` -> 直接写那件值得注意的事
- `进一步来看，……` -> 直接写下一层结果

## 7.5 带读者读 -> 直接写图和结果

- `放在一起看，图 2 最需要盯住的是……`
  -> `图 2 里最明显的是……`

- `读这篇文章时，有两个特点要先抓住。`
  -> 直接写那两个特点，或者只保留真正需要的那一个

- `这张图的作用很朴素，就是告诉你……`
  -> `这张图显示的是……`

## 8. 指标先报数后上课 -> 报数时就解释

- `男性中胸痛 OR 2.2。这里的 OR 可以理解成……`
  -> `男性里胸痛的 OR 是 2.2，也就是在同样已经叫了急救的人里，后来发生心脏骤停的那组更常提到胸痛。`

- `HR 1.35。也就是说风险升高了 35%。`
  -> `HR 1.35，也就是随访期间这一组的事件发生速度大约高出 35%。`

## 9. 翻译腔长句 -> 两步拆开

原句常见问题：

- 修饰语太多
- 主语太晚出现
- 一句里塞背景、方法、结果、边界

改法：

1. 先把主语和动作拉到前面
2. 再把补充信息分到下一句

例子：

- `基于长期随访队列的深度表型数据的风险关联分析显示……`
  -> `这项分析用的是长期随访队列的深度表型数据。结果显示……`

## 10. 加一点人味，但不要过头（不适用于论文 / 医学 / 学术正文）

适合少量用：

- `我更在意的是……`
- `我会这样理解……`
- `其实……`
- `说实话，……`
- `有点可惜的是……`

不适合反复用：

- 每段都 `我觉得`
- 每段都 `其实`
- 靠 `说白了 / 这事儿 / 你会发现` 刷存在感

判断标准：

- 这句话加了之后，是更像人在说话，还是更像在表演“像人”？

## 10.5 读者管理口气 -> 平视表达

- `普通人最该记住的是，一旦出现这些信号别往后拖。`
  -> `这些信号出现以后，后续处理需要尽快接上。`

- `最需要盯住的是……`
  -> `更该注意的是……`
  -> 或者直接删掉前半句，落事实本身

- `不能轻轻放过`
  -> `需要尽快处理`

## 11. 过于平滑的句群 -> 打开一点节奏

- `变量之间差异很大，缺失并不随机，结局定义也常常不完全一致，再加上志愿者偏倚和族群结构偏倚……`
  -> `这类数据的问题不少。变量之间差异很大，缺失也不随机；结局定义常常不完全一致，志愿者偏倚和族群结构偏倚也一直都在。`

改法重点：

1. 先给一句短的定点句
2. 再展开，不要所有项都挤在同一口气里
3. 不用把每个并列项都打磨成同样重、同样满

## 12. 解释过满 -> 留下最重要的一步

- `如果没有一套统一的数据治理和评价办法，即使结果显著，后面也很难留下可复用、可核对的证据链。`
  -> `如果没有同一套数据治理和评价规则，结果就算显著，后面也很难留下能反复核查的证据。`

改法重点：

1. 只留最重要的后果
2. 砍掉重复修饰
3. 别把每一层逻辑都补成满分答案

## 12.5 判断收得太满 -> 留一点自然空白

- `这些队列提供了大样本、多维度和长期随访的数据条件，使研究者能够……并把研究问题从……推进到……`
  -> `这些队列让研究者第一次能在同一批人身上长期看这么多东西。研究问题也跟着变了，不再只是盯着单一疾病和少数变量。`

改法重点：

1. 别把“条件、过程、结果、意义”一次性全写满
2. 先把最想说的判断落下来
3. 后面那层解释如果只是把意思再说一遍，就删掉

## 12.8 两段式综述推进太标准 -> 打破同一套段落任务

常见问题：

- 第一段讲数据基础
- 第二段讲方法发展
- 两段都按“铺垫 -> 解释 -> 收束”推进

改法思路：

1. 让其中一段直接落在问题上
2. 让另一段直接落在限制或冲突点上
3. 不要两段都写成完整的小综述

例子：

- `近几年……先是数据基础建设。……第二条是模型发展。……`
  -> `近几年，深度表型研究先把数据底子搭起来了。……另一头，表示学习和医学基础模型也在往前走。可问题很快就出来了：……`

这样改的重点不是“更口语”，而是让两段承担不同任务，不再像标准模板并排展开。

## 13. 隐性连接太密 -> 直接落事实

- `不过，这类数据的麻烦也一直都在。`
  -> `这类数据的麻烦一直都在。`

- `但这里也有一个很现实的问题：`
  -> `这里有一个很现实的问题。`

改法重点：

1. 能不用“不过 / 但这里也有”就不用
2. 直接把句子落在问题本身

## 14. 标准化术语组合 -> 拆开重写

- `统一的数据治理和评价办法`
  -> `同一套数据治理和评价规则`

- `可复用、可核对的证据`
  -> `后面还能复用、也能对照核查的证据`

- `统一表示方法`
  -> `能把这类数据放进同一套表示里处理的方法`

- `缺失语义和无序输入一起处理好`
  -> `既能看懂缺失本身的含义，也能处理这类没有固定顺序的输入`

改法重点：

1. 拆掉模板短语
2. 落回动作、对象、用途

## 15. 产品 / PRD / README 黑话 -> 写清对象、动作、结果

- `我们需要把投放、转化和留存放到一个统一矩阵里看。`
  -> `我们需要把投放、转化和留存放到同一张表里一起看。`
  -> `我们需要把这三组指标放在一起比较。`

- `先把用户链路拉通，再沉淀一套打法。`
  -> `先把用户从进入到转化的流程接起来，再把有效做法整理下来。`

- `这次优化的核心抓手，是把运营动作和产品触达做成闭环。`
  -> `这次优化的重点，是让运营动作和产品触达前后对应起来。`
  -> `这次优化的重点，是让运营动作发出去以后，后面的产品触达能接上。`

- `把权限颗粒度再细一点。`
  -> `把权限继续往下拆一层。`
  -> `把权限分得更细一点。`

改法重点：

1. 不要只把黑话换成另一组黑话
2. 能直接写对象、动作、前后关系，就直接写
3. README 和 PRD 优先追求清楚，不追求“管理感”
