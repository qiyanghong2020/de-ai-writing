# 论文 / 医学写作专用参考

这份参考不是通用润色清单，而是给中文论文、学位论文、医学研究写作用的。

硬规则先放前面：

- 不要把 `研究表明 / 数据显示 / 结果提示 / 相关分析发现` 这类证据报告词改成口头观察句
- 不要为了“更自然”机械把 `机制`、`框架`、`临床层面` 这类术语降成口语
- 不要把结果段改成评论体或导读体

重点不是“句子顺不顺”，而是：

- 这个词像不像该领域的人会这样写
- 这个句子像正文、像图注、还是像项目汇报
- 这个表述是在说结果，还是在说作者如何组织结果

## 快速目录

- `1. 先分清段落类型`：标题、方法、结果、讨论/结论的语域差异。
- `2. 医学论文里常见的“顺但不稳”词`：证据链、闭环、框架、主线、支撑、路径等。
- `2.1. 医学 / 生物论文里的英文表格也要查语域`：中文任务中夹带英文医学表格时的转交和保护规则。
- `2.5. 不只是坏词，还要防五种“机器感”`：转场、句式、语法、词汇和作者立场。
- `3-5`：不要机械替换、实用替换模式、按章节复查。
- `6-8`：博士论文人工痕迹、最后自检、长文执行方式。

## 1. 先分清段落类型

同一个词，在不同位置可接受程度不同。

### 标题 / 小标题

标题要稳，不要太像产品 roadmap 或咨询框架。

高风险：

- `验证路线图`
- `技术闭环`
- `工程化交付物`
- `统一主线`
- `转化路径`

优先改法：

- `验证设计`
- `技术安排`
- `输出材料`
- `主要脉络`
- `转化方向`

### 方法段

方法段允许较强的结构词，但要避免产品化或管理化。

高风险：

- `证据链整合`
- `闭环`
- `交付物`
- `口径表`
- `核对`

优先改法：

- `结果整合`
- `前后对应`
- `材料`
- `统一说明表`
- `复核` / `比较` / `一致性检查`

### 结果段

结果段应尽量直写观察到的现象和证据，不要用“组织结果的词”代替结果本身。

高风险：

- `形成证据链`
- `结构性结果`
- `支撑性结论`
- `线索`
- `闭环`

优先改法：

- `形成相互印证的结果`
- `分项结果` / `分层结果`
- `辅助性结果`
- `依据` / `特征` / `观察结果`
- `相互印证`

额外高风险：

- `结果框架`
- `结果整合`
- `结果组织`
- `双重结果表达`
- `可复核的整体结果`

更稳的方向：

- 直接写 `各部分结果相互印证`
- `综合评估`
- `同时保留结构图和统计摘要`
- `形成总体结论`
- `将结果分为……`

### 讨论 / 结论段

讨论段最容易出现“看起来高级但不贴领域”的抽象词。

高风险：

- `证据链`
- `主线`
- `底座`
- `补齐`
- `实现路径`

优先改法：

- `证据基础` / `相互印证的结果`
- `主要脉络`
- `共同基础`
- `补充`
- `实现方案`

额外高风险：

- `主结论`
- `辅助性结果`
- `支撑……`
- `补充支持`
- `一致结果框架`
- `多任务研究框架`

更稳的方向：

- `最强证据来自……`
- `补充结果` / `补充证据`
- `用于……` / `可用于……`
- `一致的结果`
- `统一的多任务分析方案`

## 2. 医学论文里常见的“顺但不稳”词

这些词不是绝对错，但在医学论文里常比作者预期更像项目汇报。

### `证据链`

问题：

- 太容易带出咨询、产品、工程汇报口气
- 一段里反复出现时，像在管理论证，而不是在陈述结果

可替换为：

- `证据基础`
- `各部分结果之间的对应关系`
- `证据体系`
- `相互印证的结果`
- 直接写具体层级：`分型结果、纵向支持和外部比较`

### `闭环`

问题：

- 太像互联网产品、流程管理和工程复盘

可替换为：

- `前后对应`
- `相互印证`
- `形成较完整的证据基础`
- `使方法与结果能够对应`

### `交付物`

问题：

- 太像工程项目或产品团队语境

可替换为：

- `材料`
- `输出材料`
- `补充材料`
- `模型说明、字段说明和复现脚本`

### `核对`

问题：

- 日常中文没问题，但在正式医学论文里常偏口语或偏流程说明

可替换为：

- `复核`
- `比较`
- `一致性检查`
- `审计`

选择规则：

- 对数字、字段、样本量：`复核`
- 对方向、排序、外部支持：`比较` / `一致性检查`
- 对来源和数据流程：`审计`

### `线索`

问题：

- 真正探索性语境可以用
- 如果结果已经较明确，再写 `线索` 会显得含混

可替换为：

- `依据`
- `特征`
- `候选因素`
- `观察结果`

选择规则：

- 真正假设生成：可以保留 `线索`
- 已有明确方向或量化支持：优先 `依据` / `特征`

### `框架`

问题：

- 在论文里本来可以用，但过密时会把正文写成“管理研究结构”而不是“陈述研究对象”
- 在结果段和讨论段尤其容易显得过于 schematic

可替换为：

- `评价标准`
- `验证过程`
- `验证依据`
- `统一方法`
- `综合评估`
- 直接改写整句，不保留 `框架`

选择规则：

- 方法总述或固定术语：可以保留
- 结果段、讨论段、总结段：优先降抽象度

### `主线` / `主轴`

问题：

- 很像综述评论、咨询文档或答辩提纲

可替换为：

- `研究方向`
- `主要脉络`
- `全文围绕……展开`
- `全文主要讨论……`

### `支撑`

问题：

- 论文里不是错，但高频出现时很像项目汇报
- 尤其在 `支撑任务`、`支撑结论`、`支撑策略` 这类搭配里显得抽象

可替换为：

- `用于`
- `可用于`
- `说明`
- `形成`
- `提供依据`
- `得到`

例子：

- `支撑疾病分型、共病网络和风险预测` -> `用于疾病分型、共病网络分析和风险预测`
- `支撑筛查与随访优先级策略` -> `为筛查与随访优先级提供依据`
- `主结论主要由……支撑` -> `主结论主要依据……` / `最强证据来自……`

### `路线` / `路径`

问题：

- 在论文标题、小节名和讨论段里，容易变成 roadmap 口气

可替换为：

- `方向`
- `步骤`
- `思路`
- `安排`
- `后续验证步骤`

### `结果`

问题：

- `结果` 本身当然能用
- 但一旦和 `框架 / 整合 / 组织 / 表达` 连用，常常是在“解释作者怎么摆结果”，而不是在讲结果本身

高风险组合：

- `结果框架`
- `结果整合`
- `结果组织`
- `结果表达`
- `可复核的整体结果`

优先改法：

- 直接写 `各部分结果相互印证`
- `综合评估`
- `结果呈现`
- `总体结论`
- `将结果分为……`

### `工程化` 相关词

高风险：

- `工程化输出材料`
- `工程体系`
- `工作流`
- `下游接口`
- `表征接口`

问题：

- 太像系统设计、研发复盘或产品交付语境

可替换为：

- `补充材料`
- `实现基础`
- `研究流程`
- `下游任务应用`
- `同一表征用于……`

## 2.1. 医学 / 生物论文里的英文表格也要查语域

本参考主要处理中文论文和中文医学写作。若完整文稿是英文医学/生物论文，尤其是英文表格、表头、biomarker 表格或 validation table，应转到 `de-ai-writing/references/english.md` 的 `Scientific and medical table language`。

若中文论文或中文审稿任务中夹带英文表格，只做语域提醒：表头应像 evidence table、biomarker table 或 validation table；避免内部框架标签和项目管理式分类；不要机械替换，不要改变列数、统计量、单位、P 值、CI、HR/OR/RR、基因符号大小写、蛋白名和斜体/非斜体约定。`clinical utility`、`clinical actionability` 和 `predictive utility` 有证据门槛，早期关联或机制研究优先用 `clinical relevance`、`potential clinical implication` 或 `translational relevance`。

### `提醒`

问题：

- 偏口语、偏综述导读语气

可替换为：

- `提示`
- `表明`
- `说明`

## 2.5. 不只是坏词，还要防五种“机器感”

很多论文段落不是词错了，而是写法太整齐。

### A. 转场词过密

高风险：

- `从方法学上看`
- `从科学问题看`
- `从临床和公共卫生角度看`
- `综上`
- `据此`
- `为此`
- `换句话说`
- `按证据强度看`

问题：

- 像讲义或答辩稿在切换视角
- 先提醒读者“我要怎么说”，再说内容本身

优先改法：

- 能删就删
- 直接从对象、动作、结果起句
- 若必须承接，只保留最短的因果或让步关系

例子：

- `从方法学上看，本文关心的不是……` -> `本文关心的不是……`
- `综上，……` -> 直接写总结句
- `为此，本文补充……` -> `本文补充……`

### B. 句式骨架重复

高风险：

- `第一方面……第二方面……第三方面……`
- `第一……第二……第三……`
- `其一……其二……`
- `在疾病分型任务中……在共病网络分析中……在疾病预测任务中……`
- `若……则……`
- `本文……并……`
- `这样……也……`

问题：

- 单句没错，但连续出现时很像模板生成
- 段落之间的起势、节奏和论证动作都一样

优先改法：

- 打破连续平行句
- 把三段里至少一段改成不同起句
- 把列表改写成“总句 + 分号”或“总句 + 两句展开”
- 不要每一段都先交代写作结构

例子：

- `第一方面……第二方面……第三方面……` -> `最先成熟的是……另一个重要方向是……与此并行的，是……`
- `在疾病分型任务中……在共病网络分析中……在疾病预测任务中……` -> `疾病分型部分……共病网络部分……风险预测部分……`

### C. 语法太完美

高风险表现：

- 抽象名词连续堆叠
- 一段里同时完成“总括、限定、分层、边界声明”
- 每句都在承担一个很整齐的论证功能
- 读起来没有卡顿，但也没有自然起伏
- 句子都很正式、很完整、很稳妥，像统一抛光过
- 相邻句子的长度、力度和论证动作都太接近

常见区域：

- 中文摘要结果段
- 绪论里的研究意义、综述、研究缺口
- 结果章末的小结
- 讨论和结论

优先改法：

- 允许句子长短不完全对齐
- 把过度平衡的三层、四层结构打散
- 少用“这项工作的主要贡献在于……”这类 polished summary
- 能直接说“最强证据来自……”就不要先搭一个结论分层舞台
- 留下少量朴素直写句，不要每句都写成正式综述句
- 同一段里允许一两句只做观察、比较或限定，不必句句都承担完整论证

硬限制：

- 这里说的“自然的人性化不完美”不是故意留下病句
- 不要为了去 AI 腔引入语法错误、口语病或逻辑松散
- 真正要恢复的是自然起伏、轻重变化和局部不完全对称

### D. 词汇太平

高风险表现：

- 相邻几段反复使用同一组安全词，如 `结果`、`问题`、`分析`、`框架`
- 解释性动词过于单一，如连续使用 `提示`、`表明`、`说明`
- 评价词过于整齐，如连续出现 `稳定`、`一致`、`显著`
- 词都不算错，但放在一起像“稳妥通用稿”，不像这篇论文自己的语言

常见区域：

- 中文摘要
- 绪论综述
- 结果章小结
- 讨论和结论

优先改法：

- 按段落功能换词，而不是机械找同义词
- 结果段多用观察和比较词，如 `高于`、`低于`、`集中在`、`分布于`
- 方法段多用研究动作词，如 `纳入`、`设定`、`比较`、`复核`
- 讨论段保留 `提示`、`说明`、`仍需验证` 这类词，但不要一段内反复堆叠
- 不要为了“多样”硬换词，先保证精确，再追求不呆板

### E. 作者立场感太弱

这里说的不是轶事，也不是抒情，而是学术写作里应有的作者选择和判断。

高风险表现：

- 每句都完全无人称、无立场，像任何一篇论文都能套用
- 只在搬运事实，不交代本研究真正关心什么、刻意不做什么、最强证据在哪里
- 整段很稳，却没有“这段为什么要这样写”的作者判断

优先改法：

- 允许克制的作者立场，如 `本文更关心的是……`
- 可以直接说明边界，如 `本研究不打算回答……`
- 可以点出重点，如 `这里更直接的问题是……`
- 可以明确权重，如 `最强证据来自……`
- 仍然不要引入个人经历、轶事或随笔式插话

## 3. 这类词不要机械替换

同一个词，有时该改，有时不该改。

### 可以保留的情况

- 图题、表题、自动目录、自动域
- 文献题名原文
- 真正探索性结论中的 `线索`
- 该领域确实常见且不显突兀的固定说法

### 应优先改写的情况

- 同一页出现 3 次以上类似抽象词
- 句子主要在管理论证，而不是讲对象、动作和结果
- 读起来更像“介绍这篇论文怎么组织”，而不是“论文发现了什么”
- 词语放进别的行业文档也一样成立

## 4. 实用替换模式

不要只替换一个词，优先一起改整句骨架。

### 模式 A

- 原：`本文形成了分型—网络—预测的一致性证据链`
- 改：`本文在同一底层表示下同时完成分型、网络和预测分析，各部分结果相互印证`

### 模式 B

- 原：`这些结果构成了较完整的闭环`
- 改：`这些结果相互印证，构成了较完整的证据基础`

### 模式 C

- 原：`形成模型卡、口径表和复现脚本的交付物`
- 改：`同步整理模型说明、字段说明和复现脚本`

### 模式 D

- 原：`对共享风险因子进行核对`
- 改：`对共享风险因子进行比较`

### 模式 E

- 原：`把这些结果转成可验证的分层线索`
- 改：`把这些结果转成可验证的分层依据`

### 模式 F

- 原：`全文主轴始终围绕一个问题展开`
- 改：`全文围绕一个问题展开`

### 模式 G

- 原：`支撑疾病分型、共病网络和风险预测`
- 改：`用于疾病分型、共病网络分析和风险预测`

### 模式 H

- 原：`本文将第 5 章的风险预测结果组织成……`
- 改：`第 5 章的风险预测结果分为……`

### 模式 I

- 原：`形成可解释、可复核的结果框架`
- 改：`得到可解释、可复核的结果`

### 模式 J

- 原：`未来工作路线可沿……四条主线并行推进`
- 改：`未来工作可沿……四个方向推进`

## 5. 按章节再看一遍

除了看词，还要看这些词出现在哪一章。

### 摘要

- 最怕 `结果框架`、`支撑……任务`、`复现框架`
- 也要警惕 `方法上……随后……接着……为便于……` 这种输送带式推进
- 摘要应尽量直接写：做了什么、得到什么、边界在哪里

### 绪论 / 综述

- 最怕 `主线`、`判断框架`、`分析坐标`、`路线图`
- 也要警惕连续几个段落都用 `从X看`、`第一方面/第二方面/第三方面`
- 综述段更适合 `研究方向`、`主要脉络`、`评价标准`

### 方法章

- 最怕 `工作流`、`接口`、`工程化`、`底层方法`
- 方法章可以保留结构词，但要更像研究流程，不要像系统设计说明

### 结果章

- 最怕 `结果框架`、`结果组织`、`双重结果表达`、`结构性解读`
- 也要警惕连续平行写法，如 `在分型任务中……在网络任务中……在预测任务中……`
- 结果章优先讲观察、比较、差异、验证，不要讲“我怎么摆这些结果”

### 讨论 / 结论

- 最怕 `主结论由……支撑`、`一致结果框架`、`多任务研究框架`、`补充支持`
- 也要警惕 `按证据强度看，全文结论可以分成三层` 这类过于规整的总结段
- 讨论章更适合写 `最强证据来自……`、`当前结果能够说明……`、`仍需进一步验证……`

## 6. 博士论文更像人工写作，不靠华丽，靠真实研究痕迹

一句话判断：

- 更像人工主导完成的博士论文，往往让人感觉“这个人真的做过这件事”
- 更像机器整理的论文，往往让人感觉“这段文字知道这个题应该怎么被说出来”

### 6.1 问题要有来路

高价值特征：

- 研究问题能从前人工作、数据结构、方法缺口或现实限制里长出来
- 不是只说“这个问题重要”，而是说清为什么这个问题在这里必须这样问
- 问题定义有明确边界，不是一路外推到很大的结论空间

优先改法：

- 少写标准背景总述，多写这个问题为何在当前材料、当前方法、当前任务下成立
- 能点出真正的难点，就不要只写泛化的“仍有局限”

### 6.2 综述要有取舍，不要平均用力

高价值特征：

- 真正相关的工作讲得更具体
- 只是铺垫的工作可以一带而过
- 会指出已有工作的具体不足，而不是统一写成“仍有局限”
- 引文取舍能看出作者的方向、训练背景和学术立场

高风险：

- 每篇文献都像差不多重要
- 综述像平铺信息，而不是为问题定义服务
- 引言连续按 `数据基础建设 -> 方法发展 -> 应用拓展 -> 文献缺口` 这类对称链条展开，过于完整、过于标准
- 章节起句像模板说明，如 `本章交代……并给出……为后续章节……`，更像在介绍写作安排，而不是进入研究问题

### 6.3 方法要留下真实研究摩擦

这里的“摩擦”不是故事感，而是研究现场里的真实约束。

高价值特征：

- 为什么采用当前方案，而不是另一个看起来也合理的方案
- 预处理、采样、质控、缺失处理、编码方式这些地方更具体
- 能交代局部限制，如样本太少、编码会引入偏差、某策略在特定分布下不稳定
- 会区分 benchmark 上表现好，和临床解释性或可转化性更强，不把两者混为一谈

硬限制：

- 可以保留源文本里已有的摩擦和局部限制
- 不要凭空捏造“试过哪些方案”“后来放弃了什么”

### 6.4 结果要有主次，不要把所有结果写成一个音量

高价值特征：

- 哪个是主结果，哪个是支持性结果，哪个结果虽然显著但不够稳，要看得出来
- 结果像证据推进，而不是结论陈列
- 不同结果承担的作用不同：主发现、补充比较、稳健性检查、边界说明

优先改法：

- 不要让每个结果都用同样完整、同样饱满的句法去写
- 允许在结果段点明权重，如 `最强证据来自……`、`这一结果更适合作为补充比较`

### 6.5 讨论要有边界，不要一路往大处吹

高价值特征：

- 同时交代贡献、局限和推进到哪一步
- 区分相关与机制
- 区分统计显著、临床意义和可转化性
- 避免把局部改进直接写成范式升级

优先改法：

- 能明确写 `当前结果能够说明……`，也要明确写 `当前结果还不能说明……`
- 能写边界，就不要只做上升总结

### 6.6 不要把全文磨成一样平

高价值特征：

- 全文主线稳，但局部并不完全均匀
- 作者最熟的部分通常更强、更具体
- 某些铺垫段可以略平，但不必和核心段落完全一个音色

硬限制：

- 这里不是要求故意留下病句、重复或错误
- 真正要避免的是把所有段落都磨成同一种安全、匀整、平均化的声调

### 6.7 保留作者判断和训练背景

高价值特征：

- 会有真正属于作者的判断句，而不是只有安全正确的话
- 能看出作者更重什么：临床问题、偏倚控制、机制解释、系统设计、推断边界
- 不把医学、工程、生物、统计这些训练背景平均化到看不见

可保留的句子类型：

- `当前方法真正卡住的不是……而是……`
- `在本数据结构下，……难以稳定恢复……`
- `某类指标提升并不意味着更好的临床决策支持`
- `该结果可能更多反映队列构成，而非疾病机制本身`

### 6.8 六个高价值关键词

- `具体`
- `取舍`
- `摩擦`
- `边界`
- `主次`
- `判断`

真正该追求的，不是“更流畅”，而是作者对问题、数据、方法、结果和局限的控制力。

## 7. 最后一轮自检

改完后，逐段问这几个问题：

1. 这段像论文作者在陈述研究，还是像项目经理在汇报进度？
2. 这个词是本领域常见叫法，还是“听起来高级”的泛词？
3. 这段是在讲数据和结果，还是在讲“我怎样组织这些结果”？
4. 如果删掉这个抽象词，直接写对象、动作和结果，会不会更清楚？
5. 这段是不是把结构管理得太好了，以至于像答辩稿或机器整理稿？
6. 相邻两三段是不是用了同样的起句或同样的句式骨架？
7. 相邻段落是不是反复依赖同一组“安全词”，以至于词汇太平？
8. 这段有没有体现作者在取舍、限定和判断，而不是只有无差别的客观陈述？
9. 这个问题是不是从前人工作、数据结构、方法缺口或现实限制里长出来？
10. 综述是不是体现了真正的取舍，而不是平均用力？
11. 方法里能不能看到真实约束、局部限制或研究摩擦？
12. 结果有没有分清主结果、支持性结果和边界性结果？
13. 讨论有没有明确说出这项工作不能说明什么？
14. 这段能不能看出作者的训练背景和判断，而不是被磨成平均化学术腔？
15. 这一段是不是过于像“标准引言模板”，只是在把背景、意义、综述、缺口按顺序铺开？
16. 章节开头是不是在介绍写作安排，而不是直接进入问题、对象或核心约束？

## 8. 长文论文的执行方式

这部分说的不是“该怎么写句子”，而是长文论文在实际处理时该怎么分批改。

### 8.1 默认先分层

论文主稿至少分成这几类：

- 正文段落
- 标题和小标题
- 图题、表题、Caption
- 自动目录、自动编号、交叉引用
- 参考文献
- 表格内容

这些区域不要默认放在同一轮里一起处理。

### 8.2 默认先诊断，再改正文

长文不建议一上来全文重写。

更稳的顺序是：

1. 找出高风险段落
2. 按章节归类问题
3. 只改正文高风险段落
4. 必要时再做相邻段落协调

高风险段优先级通常是：

- 中文摘要
- 绪论综述与研究缺口
- 结果章末的小结
- 讨论与结论

### 8.3 标题、短段和自动域默认保守

这些块默认保守处理，除非用户明确要求：

- 很短的段落
- 章节说明标签
- 图表标题
- 自动域
- 参考文献条目

原则不是“全部都改过”，而是“改真正影响文风判断的地方”。

### 8.4 每轮保留一份紧凑的风格记忆

长文处理时，不要把前面所有改写结果都当上下文反复带着走。

只需要留下少量风格记忆：

- 这一章的术语口径
- 这一章的禁用连接词
- 这一章允许的作者判断方式
- 这一章最该保住的边界
- 这一章的主要问题类型

风格记忆要短，避免把全文磨成单一音色。

### 8.5 两阶段足够，多了容易抛光过头

对大多数论文场景，更稳的是：

- 第一阶段：诊断与定向
- 第二阶段：局部重写与复核

只有在这些情况下再做第三阶段：

- 相邻段落明显脱节
- 前两轮只清掉坏词，没解决骨架
- 同一章内部仍然音色割裂

第三阶段也只做局部协调，不做全篇重新抛光。

### 8.6 借工程流程，不借“扩写伪人化”

可以借鉴的，是：

- 分段
- 跳过短段
- 分阶段
- 术语保护
- 历史压缩
- 局部复核

不该借的，是：

- 靠增加辅助词制造“人味”
- 靠扩长句子制造“解释性”
- 靠系统性替换把全文写得更满、更工整

对博士论文来说，这些往往会把文本重新推回“AI 参与后统一润色”的样子。
